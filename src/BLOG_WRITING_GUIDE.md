# WebManage Blog Writing Guide

A comprehensive, citation-backed reference for the AI blog agent. Every post generated by the agent must follow these standards. This guide synthesizes best practices from Google's official documentation, and leading SEO authorities including Semrush, Backlinko, Ahrefs, Yoast, Search Engine Journal, and HubSpot.

---

## 1. Content Philosophy: People-First, Not Search-Engine-First

Google's official guidance is unambiguous: create content for people, not algorithms. The agent must internalize this as the governing principle above all tactical SEO rules.

**Self-assessment questions from Google's helpful content guidelines:**

- Does the content provide original information, reporting, research, or analysis?
- Does it provide a substantial, complete, or comprehensive description of the topic?
- Does it provide insightful analysis or interesting information that is beyond the obvious?
- Would you bookmark this content, share it with a friend, or recommend it?
- Does the content provide substantial value compared to other pages in search results?

> Source: [Google Search Central — Creating Helpful, Reliable, People-First Content](https://developers.google.com/search/docs/fundamentals/creating-helpful-content)

**What the agent must avoid:**

- Writing content primarily to attract search traffic rather than serve the client's audience
- Summarizing competitors without adding substantial original value
- Targeting word counts based on search engine preferences rather than topic depth
- Leaving readers feeling like they need to search again to get better information
- Writing about trending topics without genuine relevance to the client's business

> Source: [Google Search Central — Creating Helpful Content](https://developers.google.com/search/docs/fundamentals/creating-helpful-content)

## 2. E-E-A-T: Experience, Expertise, Authoritativeness, Trustworthiness

Google's quality rater guidelines use E-E-A-T as the primary framework for assessing content quality. In December 2022, Google added "Experience" as the first E, emphasizing first-hand knowledge.

**How the agent should demonstrate E-E-A-T:**

- **Experience:** Reference the client's actual products, services, and real-world use cases. Use language that shows familiarity with the problem space ("If you've ever tried to..." rather than "One might consider..."). Never fabricate experiences.
- **Expertise:** Cite specific data, explain technical concepts clearly, and provide actionable depth. Avoid surface-level generalizations that any non-expert could write.
- **Authoritativeness:** Write from the client's brand perspective. The content should read as though it comes from someone established in the industry.
- **Trustworthiness:** Be factually accurate. Avoid easily-verifiable errors. When making claims, provide context or evidence.

> Sources: [Google E-E-A-T Guidelines](https://developers.google.com/search/blog/2022/12/google-raters-guidelines-e-e-a-t), [Semrush — Google E-E-A-T](https://www.semrush.com/blog/eeat/), [Backlinko — Google E-E-A-T](https://backlinko.com/google-e-e-a-t)

**AI content disclosure:** Google does not penalize AI-generated content, but it should be useful, original, and transparent. If the client chooses to disclose AI use, the content should still demonstrate genuine expertise and value.

> Source: [Google Search Central — Creating Helpful Content](https://developers.google.com/search/docs/fundamentals/creating-helpful-content)

## 3. Title Tag Optimization

The title tag is the single most impactful on-page SEO element. It appears in search results, browser tabs, and social shares.

**Rules for the agent:**

- Keep title tags under 60 characters to avoid truncation in search results
- Place the target keyword as close to the beginning as possible
- Make the title compelling — it must earn the click, not just contain the keyword
- Never use clickbait that the content doesn't deliver on
- Each title must be unique across the client's site
- Avoid filler words that add length without value ("Ultimate", "Definitive" — unless the content truly is)

**Strong title patterns:**

- How to [Action] + [Desired Outcome]: "How to Delete Photos of Your Ex Without Losing Everything"
- [Number] [Things] + [Benefit]: "5 Background Check Mistakes That Cost You Good Hires"
- [Question the reader is searching]: "Is a Background Check Required for Remote Employees?"

> Sources: [Yoast — Page Titles for SEO](https://yoast.com/page-titles-seo/), [Semrush — Title Tags](https://www.semrush.com/blog/title-tag/)

## 4. Meta Description

Meta descriptions don't directly affect rankings, but they significantly impact click-through rate (CTR), which indirectly affects rankings.

**Rules for the agent:**

- Keep under 155 characters (Google truncates beyond this)
- Include the target keyword naturally (Google bolds matching terms)
- Write it as a compelling pitch: what will the reader learn or gain?
- Use active voice and second person ("you") where appropriate
- Include a soft call-to-action ("Learn how...", "Find out why...")
- Never duplicate meta descriptions across posts
- Each description must accurately reflect the page content

> Sources: [Yoast — Meta Descriptions](https://yoast.com/meta-descriptions/), [Backlinko — Meta Descriptions](https://backlinko.com/hub/seo/meta-descriptions), [Semrush — Meta Descriptions](https://www.semrush.com/blog/meta-description/)

## 5. Heading Structure and Hierarchy

Headings create the skeleton of the post. They serve both readers (scannability) and search engines (content understanding).

**Rules for the agent:**

- **H1**: One per post. Contains the target keyword. Describes the entire post's topic.
- **H2**: Major sections. Each H2 should cover a distinct subtopic. Use 3-6 H2s per post depending on depth.
- **H3**: Subsections under H2s where needed. Use when an H2 section covers multiple sub-points.
- Never skip levels (no H1 → H3 without an H2 in between)
- Include the target keyword or close variants in at least one H2
- Include secondary keywords in other H2s and H3s where natural
- Headings should be descriptive and scannable — a reader should understand the post structure from headings alone

> Sources: [Search Engine Journal — Header Tags](https://www.searchenginejournal.com/on-page-seo/header-tags/), [Semrush — On-Page SEO](https://www.semrush.com/blog/on-page-seo/)

## 6. Keyword Strategy

Each blog post targets one primary keyword. The agent must optimize for this keyword without stuffing.

**Rules for the agent:**

- **One primary keyword per post.** Never try to rank for multiple unrelated keywords in a single post.
- Place the primary keyword in: title tag, H1, meta description, first 100 words, at least one H2, URL slug, and image alt text (if applicable)
- Use the keyword 3-5 times naturally throughout the body for a 1000-word post
- Include semantic variations and related terms (LSI keywords) to demonstrate topical coverage
- Never force keywords into sentences where they don't fit naturally
- The keyword density should feel invisible to a human reader — if it reads awkwardly, rewrite

**Keyword placement priority (highest to lowest impact):**
1. Title tag / H1
2. URL slug
3. First paragraph (first 100 words)
4. H2 headings
5. Body content
6. Meta description
7. Image alt text

> Sources: [Ahrefs — Keyword Optimization](https://ahrefs.com/blog/keyword-optimization/), [Semrush — SEO Writing](https://www.semrush.com/blog/seo-writing/), [HubSpot — Blogging for SEO](https://blog.hubspot.com/marketing/blogging-for-seo)

## 7. Content Structure and Readability

The agent writes for busy people. Every paragraph must earn its place.

**Rules for the agent:**

- **Short paragraphs:** 2-4 sentences maximum. One idea per paragraph.
- **Short sentences:** Vary length but favor concise, direct sentences. Break up any sentence over 25 words.
- **Active voice:** Use active voice as the default. Passive voice only when the actor is unknown or unimportant.
- **Second person:** Address the reader as "you" — it creates connection and clarity.
- **No fluff phrases:** Never use "In today's world...", "It goes without saying...", "As we all know...", "In this article, we will discuss...". Start with substance.
- **Transition smoothly:** Each section should flow logically from the previous one. Use transitional phrases between major sections.
- **Front-load value:** Put the most important information first in each section (inverted pyramid style). Don't bury the answer.

**Introduction rules:**
- Hook the reader in the first sentence with a problem, question, or surprising fact
- Establish what the reader will learn in 2-3 sentences
- Keep the intro under 100 words
- Include the target keyword in the first 100 words

**Conclusion rules:**
- Summarize key takeaways in 2-3 sentences
- Include a clear call-to-action relevant to the client's business
- Never introduce new information in the conclusion

> Sources: [Semrush — SEO Blog Post](https://www.semrush.com/blog/seo-blog-post/), [Backlinko — Blog SEO](https://backlinko.com/hub/content/blog-seo)

## 8. Internal Linking

Internal links connect the client's content into a topic cluster, distribute page authority, and help search engines understand site structure.

**Rules for the agent:**

- Link to 2-3 existing posts from the same client where relevant
- Use descriptive anchor text that tells the reader what they'll find (not "click here")
- Link early in the content where possible — links higher on the page carry more weight
- Link contextually from within body paragraphs (not just a list of links at the bottom)
- Never force a link where it doesn't serve the reader
- When the client has no existing posts (first post), skip internal linking — don't link to nonexistent pages

> Sources: [Semrush — Internal Links](https://www.semrush.com/blog/internal-links/), [Ahrefs — Internal Links for SEO](https://ahrefs.com/blog/internal-links-for-seo/)

## 9. Structured Data and Schema

Structured data helps search engines understand and display content in rich results (featured snippets, FAQ panels, etc.).

**Rules for the agent:**

- Include Article schema in the frontmatter metadata (title, description, author, datePublished)
- Structure content to be eligible for featured snippets: use clear question-and-answer patterns in H2s, provide concise definitions, and use lists where appropriate
- When writing how-to content, structure steps clearly and sequentially
- When writing list content, use consistent formatting for each item

> Sources: [Semrush — Schema Markup](https://www.semrush.com/blog/schema-markup/), [Backlinko — Schema Markup Guide](https://backlinko.com/schema-markup-guide)

## 10. URL Slug Optimization

The URL slug should be clean, descriptive, and keyword-rich.

**Rules for the agent:**

- Include the target keyword in the slug
- Keep slugs short: 3-5 words is ideal
- Use hyphens to separate words (never underscores or spaces)
- Remove stop words (a, the, is, of, and) unless they're part of the keyword
- Use lowercase only
- Make the slug readable and descriptive — a reader should guess the content from the URL alone

**Examples:**
- Good: `delete-photos-of-ex-iphone`
- Bad: `how-to-delete-photos-of-your-ex-from-iphone-without-losing-everything-else-complete-guide-2026`

> Sources: [Semrush — SEO Best Practices](https://www.semrush.com/blog/seo-best-practices/), [Backlinko — Blog SEO](https://backlinko.com/hub/content/blog-seo)

## 11. Word Count and Depth

Word count is not a ranking factor, but comprehensive coverage of a topic correlates with higher rankings.

**Rules for the agent:**

- Target 800-1200 words for standard informational posts
- Target 1500-2500 words for pillar/cornerstone content
- Never pad content to reach a word count — every sentence must add value
- If the topic is fully covered in 600 words, stop at 600 words
- Depth matters more than length: cover the topic thoroughly, anticipate follow-up questions, and provide actionable details

> Source: [HubSpot — Blog SEO](https://blog.hubspot.com/marketing/blog-search-engine-optimization)

## 12. AI-Specific Content Guidelines

Since this content is generated by an AI agent, these additional rules apply:

**The agent must:**
- Never fabricate statistics, studies, or quotes. If a claim needs a citation and one isn't available, use hedging language ("many experts suggest..." or "research indicates...")
- Never claim first-hand experience the client hasn't had. Write from the brand's perspective using their known expertise.
- Avoid AI-telltale phrases: "In the ever-evolving landscape of...", "Let's dive in!", "It's important to note that...", "Without further ado...", "In conclusion,..."
- Write with specificity. Replace generic statements with concrete details relevant to the client's industry.
- Vary sentence structure. AI-generated content often falls into repetitive patterns — consciously vary sentence length, structure, and opening words.
- Sound human. Read each paragraph and ask: would a knowledgeable person in this industry actually write this sentence? If not, rewrite it.

**Phrases to never use:**
- "In today's digital age/world/landscape"
- "Whether you're a [X] or a [Y]"
- "Look no further"
- "It's no secret that"
- "At the end of the day"
- "Game-changer"
- "Unlock the power of"
- "Dive deep into"
- "Navigate the complexities of"
- "Leverage" (as a verb — just say "use")
- "Streamline" (unless discussing actual process optimization)
- "Revolutionize"

## 13. Pre-Publish Checklist

Before queuing a post as `pending_review`, the agent must verify:

- [ ] Title tag is under 60 characters and contains the target keyword
- [ ] Meta description is under 155 characters and compelling
- [ ] URL slug is clean, short, and contains the target keyword
- [ ] H1 contains the target keyword naturally
- [ ] At least one H2 contains the target keyword or close variant
- [ ] Target keyword appears in the first 100 words
- [ ] Target keyword appears 3-5 times total (for ~1000 words)
- [ ] No keyword stuffing — every usage reads naturally
- [ ] Short paragraphs (2-4 sentences max)
- [ ] Active voice throughout (passive voice < 10% of sentences)
- [ ] Introduction hooks the reader and is under 100 words
- [ ] Conclusion has a clear call-to-action
- [ ] No fluff phrases or AI-telltale language
- [ ] Internal links included where the client has existing content
- [ ] Content provides genuine value — not just a rehash of what's already ranking
- [ ] Excerpt is compelling and under 200 characters
